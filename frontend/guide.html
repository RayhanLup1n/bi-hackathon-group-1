Memulai - Login & Navigasi
Cara Login
1
Buka Halaman Login
Akses aplikasi melalui browser di http://localhost:8000/login (development) atau URL yang diberikan admin untuk production.
2
Masukkan Kredensial
Isi username dan password yang telah diberikan oleh admin. Klik tombol Masuk untuk login.
3
Navigasi Menu
Setelah login, Anda akan diarahkan ke Dashboard. Menu navigasi ada di pojok kanan atas:
Dashboard - Monitoring harga real-time
Panduan - Halaman ini
Prediksi - Prediksi ML (Analyst+)
Admin - Kelola sistem (Admin only)
🔐 Menu yang tampil tergantung role Anda: Viewer hanya bisa akses Dashboard, Analyst bisa akses Prediksi, Admin bisa akses semua halaman.
2
Dashboard - Executive Decision Support
Komponen Dashboard

Dashboard adalah halaman utama untuk monitoring prioritas risiko harga pangan. Sistem menganalisis data PIHPS, cuaca, kalender hari besar, dan persebaran kota untuk menghasilkan daftar prioritas yang actionable bagi analis TPID.

📊
Stat Bar (Ringkasan)
Menampilkan 3 metrik utama: jumlah komoditas yang dipantau, jumlah risiko tinggi/kritis, dan cakupan data (persentase).
🟢
Freshness Bar
Menunjukkan kesegaran data: tanggal terakhir, umur data (hari), status layanan ML (online/offline), dan distribusi risiko (rendah, sedang, kritis). Titik hijau = data segar, kuning = agak lama, merah = sudah usang.
🔍
Search Bar
Cari komoditas, wilayah, atau risiko secara cepat. Ketik minimal 2 huruf, hasil pencarian muncul sebagai dropdown. Klik untuk membuka detail prioritas.
🏆
Prioritas Utama (Top 3)
Tiga prioritas tertinggi ditampilkan dengan kartu detail: skor prioritas, level risiko, diagnosis RCA, faktor pendukung, dan langkah selanjutnya yang direkomendasikan.
📋
Daftar Prioritas Lengkap
Seluruh daftar prioritas yang bisa difilter berdasarkan level risiko (kritis, tinggi, sedang, rendah) dan provinsi. Klik kartu untuk melihat detail.
📦
Paket Tinjauan Bersama
Sistem mengelompokkan komoditas yang sebaiknya ditinjau bersamaan (contoh: semua cabai di provinsi yang sama). Memudahkan koordinasi rapat TPID.
📑
Transparansi Data
Tombol Buka di sidebar membuka panel transparansi: daftar sumber data (PIHPS, Open-Meteo, hari besar, HET), bobot scoring prioritas, dan batasan model. Ringkas tapi penting untuk verifikasi metodologi.
🔢
Pagination Daftar Prioritas
Daftar prioritas dipaginasi agar tingginya sejajar dengan chart di sampingnya. Gunakan tombol Prev/Next atau nomor halaman untuk berpindah. Indikator menampilkan halaman aktif dan jumlah total.
Menggunakan Filter
1
Filter Level Risiko
Pilih level risiko pada dropdown di section "Daftar Prioritas Lengkap": Kritis, Tinggi, Sedang, atau Rendah. Gunakan "Reset Filter" untuk kembali ke semua data.
2
Filter Provinsi
Pilih provinsi untuk melihat prioritas spesifik wilayah. Tersedia 4 provinsi: Banten, Jawa Barat, DKI Jakarta, dan Sulawesi Selatan.
3
Export Data
Gunakan tombol CSV atau Excel di atas daftar prioritas untuk mengunduh data. Bisa juga export per-komoditas dari halaman detail.
Membaca Kartu Prioritas
🎯
Priority Score
Skor 0-100 dihitung dari 5 sinyal: posisi harga vs HET (25%), risiko forecast P90 (30%), momentum anomali (20%), persebaran kota (15%), dan cuaca + kalender (10%). Dikali faktor confidence data.
🏷️
Level Risiko
Badge berwarna menunjukkan level: RENDAH (hijau), SEDANG (biru), TINGGI (oranye), KRITIS (merah).
🔬
Diagnosis & Next Step
Setiap prioritas menunjukkan diagnosis RCA (penyebab: hari raya, cuaca, distribusi, dll.) dan rekomendasi langkah selanjutnya: Monitor, Verifikasi, Koordinasikan, atau Pertimbangkan Intervensi.
💡 Klik kartu prioritas untuk melihat detail lengkap: evidence, faktor risiko, histori harga 30 hari, opsi respons, dan form review analis.
3
Halaman Prediksi ML
Tentang Prediksi ML

Halaman Prediksi menggunakan arsitektur ML 4 lapis: Lapis 1 (LightGBM Quantile) memprediksi harga 7-14 hari ke depan, Lapis 2 (Detection) mendeteksi changepoint dan pelanggaran HET, Lapis 2.5 (FTA) mengklasifikasi ancaman penyebab (hari raya, cuaca, stok, distribusi), Lapis 3 (LLM Agent) mensintesis seluruh sinyal menjadi rekomendasi intervensi berbasis Bowtie.

Cara Menggunakan
1
Pilih Komoditas & Wilayah
Gunakan filter untuk memilih komoditas dan wilayah yang ingin diprediksi. Sistem akan menampilkan prediksi berdasarkan data historis wilayah tersebut.
2
Lihat Grafik Prediksi
Grafik menampilkan harga historis (garis solid) dan prediksi (garis putus-putus). Area berwarna menunjukkan confidence interval (rentang kemungkinan harga).
3
Baca Analisis Detection & FTA
Lapis 2 (Detection) mendeteksi lonjakan harga struktural (changepoint), pelanggaran HET, dan ketimpangan harga antar kota. Lapis 2.5 (FTA) mengklasifikasi ancaman: T1 Hari Raya, T2 Inflasi, T3 Cuaca, T4 Defisit Stok, T5 Distribusi - dengan tingkat siaga kritis/siaga/waspada/aman.
4
Lihat Rekomendasi LLM
Lapis 3 (LLM Agent) mensintesis seluruh sinyal dari forecast, detection, dan FTA menjadi rekomendasi intervensi berbasis metodologi Bowtie. Rekomendasi mencakup langkah pencegahan (preventive barriers) dan mitigasi (recovery barriers).
Memahami Output Prediksi
📉
Interval Keyakinan (Confidence Interval)
Area berwarna di sekitar garis prediksi menunjukkan rentang kemungkinan harga. Semakin lebar area, semakin tinggi ketidakpastian prediksi.
📊
Faktor Pendorong (SHAP)
Visualisasi SHAP menunjukkan fitur mana yang paling mempengaruhi prediksi harga, misalnya harga lag-7, hari raya, atau cuaca.
🔍
Sinyal Detection
Changepoint (Z-score), CUSUM drift, jarak HET, dan disparitas harga antar kota. Sinyal ini mengkonfirmasi apakah lonjakan harga bersifat struktural atau sementara.
⚠️
Ancaman FTA (T1-T5)
Klasifikasi ancaman: T1 Hari Raya, T2 Inflasi, T3 Cuaca Ekstrem, T4 Defisit Stok, T5 Gangguan Distribusi - masing-masing dengan tingkat siaga (kritis/siaga/waspada/aman).
🎯
Akurasi Model
Model dilatih dengan data historis 2020-2026. Akurasi bervariasi tergantung komoditas dan kondisi pasar. Gunakan sebagai panduan, bukan kepastian.
🤖
LLM Agent (Groq)
Rekomendasi dihasilkan oleh LLM (Llama 3.3 70B via Groq API). Agent mensintesis seluruh sinyal (forecast + detection + FTA) menggunakan metodologi Bowtie untuk menghasilkan rekomendasi intervensi yang kontekstual.
⚠️ Prediksi ML adalah alat bantu, bukan keputusan final. Selalu pertimbangkan faktor lain yang mungkin tidak tercakup dalam model.

Memahami Status & Level Risiko
Level Risiko Prioritas

Setiap komoditas-wilayah mendapat skor prioritas (0-100) yang dihitung dari 5 sinyal tertimbang. Level risiko ditentukan dari skor tersebut.

RENDAH Hijau Skor rendah
SEDANG Biru Skor menengah
TINGGI Oranye Skor tinggi
KRITIS Merah Skor sangat tinggi
Status HET (Harga Eceran Tertinggi)

HET Monitor membandingkan harga aktual dengan batas harga pemerintah:

AMAN <80% HET
WASPADA >=80% HET
KRITIS >=95% HET
MELAMPAUI >100% HET
Faktor Analisis RCA (Root Cause Analysis)

Sistem menganalisis 4 faktor secara sequential (early-exit) untuk menentukan penyebab:

D1
Check 1: Hari Raya
Apakah tanggal analisis masuk window H-14 s/d H+3 dari hari raya nasional. Jika ya: Diagnosis = DEMAND SPIKE (skip check selanjutnya).
S1
Check 2: Cuaca Ekstrem
Apakah ada cuaca ekstrem dalam 7 hari terakhir: hujan >100 mm/hari, kekeringan >14 hari, suhu >38C, atau angin >60 km/jam. Jika ya: Diagnosis = GANGGUAN SUPPLY.
S2
Check 3: Persebaran Kota
Apakah >=60% kota mengalami kenaikan serempak. Jika ya: Diagnosis = SUPPLY NASIONAL.
S3
Check 4: Kondisi Stok
Apakah stok pedagang di bawah normal (<60%). Jika ya: Diagnosis = BOTTLENECK DISTRIBUSI. Jika tidak: Diagnosis = PENYEBAB BELUM TERIDENTIFIKASI.
📌 RCA bersifat sequential: begitu satu penyebab teridentifikasi, check selanjutnya dilewati. Rekomendasi aksi disesuaikan dengan diagnosis dan level confidence data.
5
Halaman Admin (Khusus Admin)
Akses Admin

Halaman Admin hanya bisa diakses oleh user dengan role Admin. Di sini Anda bisa melihat analisis FTA & Bowtie secara real-time dan memantau hasil inference ML.

Fitur Admin
🌳
FTA Dashboard
Visualisasi Fault Tree Analysis yang menampilkan semua ancaman (threat) yang sedang aktif. Setiap ancaman ditampilkan dengan status dan threshold-nya.
🎀
Bowtie Analysis
Menampilkan barrier pencegahan dan mitigasi untuk setiap ancaman. Membantu menentukan respons yang tepat berdasarkan kondisi saat ini.
🤖
ML Inference Real-time
Menampilkan hasil inference dari ML Engine secara real-time. Termasuk prediksi harga, confidence level, dan rekomendasi dari LLM Agent.
📊
System Status
Monitoring status sistem: koneksi database, ML Engine, dan API health. Membantu troubleshooting jika ada masalah teknis.
🔒 Halaman Admin berisi informasi sensitif dan kontrol sistem. Pastikan hanya user yang berwenang yang memiliki akses Admin.
6
Tips & FAQ
Tips Penggunaan
🔍
Gunakan Search untuk Navigasi Cepat
Ketik nama komoditas atau wilayah di search bar Dashboard untuk langsung menuju detail komoditas. Minimal 2 karakter untuk memulai pencarian.
📊
Export Data untuk Rapat
Klik tombol Export di Dashboard untuk download CSV seluruh daftar prioritas, atau Export di halaman detail untuk satu komoditas-wilayah. Cocok untuk bahan presentasi atau diskusi tim.
📱
Akses Mobile
Aplikasi responsive dan bisa diakses dari smartphone. Cocok untuk monitoring cepat saat di lapangan.
🔄
Data Freshness
Data PIHPS diperbarui setiap hari kerja melalui pipeline ETL otomatis (pukul 07:00 WIB). Cek freshness bar di atas Dashboard untuk melihat kapan data terakhir diperbarui. Data tidak tersedia untuk hari libur dan weekend.
Pertanyaan Umum (FAQ)
?
Mengapa saya tidak bisa akses halaman tertentu?
Platform menggunakan RBAC (Role-Based Access Control). Halaman Prediksi dan FTA/Bowtie hanya bisa diakses oleh Analyst atau Admin. Halaman Admin hanya untuk user dengan role Admin. Hubungi admin untuk upgrade akses jika diperlukan.
?
Dashboard menampilkan "Data terakhir: X hari lalu" - apakah itu normal?
Data PIHPS tidak tersedia untuk hari libur dan weekend. Jika hari ini Senin, data terakhir mungkin dari hari Jumat (3 hari lalu). Ini normal. Jika freshness lebih dari 3 hari kerja, hubungi admin untuk cek pipeline ETL.
?
Apa bedanya level risiko dengan status HET?
Level risiko (rendah/sedang/tinggi/kritis) dihitung dari composite scoring 5 sinyal: posisi harga, forecast, momentum, persebaran, dan cuaca/kalender. Status HET secara spesifik membandingkan harga aktual vs Harga Eceran Tertinggi pemerintah. Keduanya saling melengkapi untuk memberikan gambaran lengkap.
?
Prediksi ML tidak akurat?
Prediksi ML adalah estimasi berdasarkan pola historis. Faktor eksternal seperti kebijakan mendadak atau bencana mungkin tidak tercakup dalam model. Selalu gunakan bersama analisis RCA dan konteks lapangan.
?
Bagaimana cara melaporkan bug?
Hubungi tim teknis melalui admin atau email ke tim pengembang. Sertakan screenshot dan langkah untuk mereproduksi masalah.
📞 Butuh bantuan lebih lanjut? Hubungi tim support atau lihat API Documentation untuk integrasi teknis.
