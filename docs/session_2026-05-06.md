# Session Log — 2026-05-06

## Ringkasan
Implementasi sistem autentikasi pengguna (login, JWT, manajemen user) untuk RCA RadarPangan.

---

## Yang Dibangun

### 1. Auth Database (`src/data/auth_db.py`)
- SQLite baru di `data/auth.db`
- Tabel `users`: `id`, `username`, `password_hash`, `role`, `created_at`
- Password di-hash pakai `bcrypt` langsung (tanpa passlib)
- Auto-seed 2 akun default saat pertama kali server jalan:
  - `admin` / `admin123` → role `admin`
  - `analyst` / `analyst123` → role `analyst`
- Fungsi: `init_db_auth`, `get_user_by_username`, `verify_password`, `list_users`, `create_user`, `update_user`, `delete_user`

### 2. Auth API (`src/api/auth_routes.py`)
- Prefix: `/api/auth`
- JWT HS256, expire 8 jam, secret di `_SECRET` (ganti di production)
- Endpoint:
  - `POST /api/auth/login` — form login, kembalikan JWT + user info
  - `GET  /api/auth/me` — data user aktif (butuh Bearer token)
  - `GET  /api/auth/users` — daftar semua user (admin only)
  - `POST /api/auth/users` — tambah user (admin only)
  - `PATCH /api/auth/users/{id}` — edit password/role (admin only)
  - `DELETE /api/auth/users/{id}` — hapus user (admin only)
- Role yang tersedia: `admin`, `analyst`, `viewer`

### 3. Login Page (`frontend/login.html`)
- Tersedia di `http://localhost:8000/login`
- Glassmorphism design matching dashboard utama
- Show/hide password toggle
- Redirect otomatis ke dashboard jika sudah login
- Tampilkan akun demo di bawah form

### 4. Admin Panel (`frontend/admin.html`)
- Tersedia di `http://localhost:8000/admin`
- Hanya bisa diakses role `admin` (guard client-side via `/api/auth/me`)
- Tabel user: ID, Username, Role badge, Tanggal dibuat, Aksi
- Modal tambah user (username + password + role)
- Modal edit user (password opsional + role)
- Modal konfirmasi hapus
- Toast alert sukses/error

### 5. Update `main.py`
- Import dan register `auth_router`
- Tambah `init_db_auth()` di lifespan
- Route baru: `GET /login`, `GET /admin`

### 6. Update `frontend/index.html`
- Auth guard di awal script: cek token via `/api/auth/me`, redirect ke `/login` jika tidak valid
- `fetchJSON` sekarang kirim `Authorization: Bearer <token>` di setiap request
- Auto-logout jika API kembalikan 401
- Header: tampil nama + role user, tombol 👥 Users (admin only), tombol Keluar

## Known Issues / Catatan
- `SECRET_KEY` di `auth_routes.py` masih hardcoded — **ganti dengan env variable di production**
- Guard di `/admin` dan `/` bersifat client-side (JWT check via API) — bukan server-side redirect
- `passlib` dihapus karena tidak kompatibel dengan `bcrypt >= 4.0`
