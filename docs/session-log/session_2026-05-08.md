# Session Log тАФ 2026-05-08

## Ringkasan
Penyempurnaan sistem autentikasi: role-based routing, pemisahan dashboard per role, rename UI, dan auth guard untuk semua halaman.

---

## Yang Dikerjakan

### 1. Fix Kompatibilitas `bcrypt` + `passlib`
- **Masalah**: `passlib 1.7.4` tidak kompatibel dengan `bcrypt >= 4.0` (error saat startup)
- **Solusi**: Hapus `passlib`, pakai `bcrypt` langsung di `auth_db.py`
- `requirements.txt`: ganti `passlib[bcrypt]==1.7.4` тЖТ `bcrypt>=4.0.0`
- `auth_db.py`: ganti `CryptContext` dengan fungsi `_hash()` dan `_verify()` native bcrypt

### 2. Role-Based Dashboard (Pemisahan Tampilan)
Berdasarkan role, dashboard `/` menampilkan fitur yang berbeda:

| Fitur | Admin | Analyst | Viewer |
|-------|-------|---------|--------|
| Dashboard RCA | тЬУ | тЬУ | тЬУ |
| Jalankan RCA | тЬУ | тЬУ | тЬУ |
| Simulasi Tanggal | тЬУ | тЬУ | тЬЧ |
| Pantau Database | тЬУ | тЬУ | тЬУ |
| Tombol ЁЯСе Users | тЬУ | тЬЧ | тЬЧ |
| Halaman `/admin` | тЬУ | тЬЧ | тЬЧ |

Implementasi di `index.html` тАФ auth guard membaca role dari `/api/auth/me` lalu:
- `admin` тЖТ tampilkan tombol ЁЯСе Users
- `viewer` тЖТ sembunyikan card Simulasi Tanggal (`#simCard`)

### 3. Login Redirect тАФ Semua Role ke Dashboard Dulu
- Sebelumnya: admin login тЖТ langsung ke `/admin`
- Sekarang: **semua role login тЖТ ke `/` (dashboard)** terlebih dahulu
- Admin tetap bisa akses `/admin` via tombol ЁЯСе Users di header
- Alasan: admin juga perlu lihat data RCA, bukan langsung ke manajemen user

### 4. Rename "DB Debug" тЖТ "Pantau Database"
- Tombol di `index.html` header: `DB Debug` тЖТ **`Pantau Database`**
- Title dan header halaman `debug.html`: `DB Debug Viewer` тЖТ **`Pantau Database`**
- Halaman ini bisa diakses **semua role** (bukan hanya admin)

### 5. Auth Guard di `debug.html`
Sebelumnya `debug.html` bisa diakses tanpa login. Sekarang:
- Cek token via `/api/auth/me` saat load тАФ redirect ke `/login` jika belum login
- `fetchJSON()` di debug.html sekarang kirim `Authorization: Bearer <token>`
- Nama + role user ditampilkan di header kanan halaman
